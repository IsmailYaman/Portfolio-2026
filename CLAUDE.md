## Architecture Overview

### Tech Stack
- **Framework**: TanStack Start (React SSR framework built on TanStack Router)
- **Runtime**: Bun (development & production)
- **UI Components**: Shadcn UI (Radix UI + Tailwind CSS v4)
- **Styling**: Tailwind CSS v4 with custom theme configuration
- **Backend**: Appwrite (authentication & storage)
- **State Management**: TanStack Query for server state
- **Animation**: Motion library (Framer Motion successor)

### Routing Architecture

This app uses TanStack Router with file-based routing. Route files live in `src/routes/`.

**Route Layouts (Layout Routes):**
- `__root.tsx` - Root layout with theme provider, global meta tags, auth middleware
- `_public.tsx` - Public routes (no authentication required)
- `_auth.tsx` - Auth pages (redirects logged-in users away)
- `_protected.tsx` - Protected routes (requires authentication)

**How Route Protection Works:**
1. `__root.tsx` loader runs `authMiddleware()` on every request to get current user
2. `_protected.tsx` loader checks if user exists, redirects to `/sign-in` if not
3. `_auth.tsx` loader checks if user exists, redirects to `/` if they're already logged in

Routes are automatically typed via `src/routeTree.gen.ts` (generated by `tsr generate`).

### Server-Side Architecture

**Server Functions** (`src/server/functions/auth.ts`):
- Uses `createServerFn()` from TanStack Start for type-safe server actions
- Server functions run only on the server and can access server-only APIs (Appwrite SDK)
- Client calls server functions like regular async functions with full type safety

**Authentication Flow:**
1. User submits credentials to `signInFn` or `signUpFn`
2. Server function creates Appwrite session using admin client
3. Session stored in HTTP-only cookies (`appwrite-session-secret`, `appwrite-session-id`)
4. Subsequent requests use `createSessionClient()` with session cookie
5. `getCurrentUser()` validates session and returns user or null

**Appwrite Clients** (`src/server/lib/appwrite.ts`):
- `createAdminClient()` - Uses API key for user creation, password recovery
- `createSessionClient(session)` - Uses session cookie for authenticated requests
- Never import these in client code (server-only)

### Password Recovery Flow
1. User submits email at `/forgot-password` → `forgotPasswordFn()`
2. Appwrite sends email with recovery link to `/reset-password?userId=X&secret=Y`
3. User clicks link, enters new password → `resetPasswordFn()`
4. User redirected to `/sign-in` to use new credentials

The recovery URL is auto-detected from the `origin` header, so it works across environments.

### Custom Production Server

`server.ts` implements a high-performance Bun production server with:
- Intelligent asset preloading (small files in memory, large files on-demand)
- Configurable via environment variables (see `server.ts` header comments)
- ETag support for cache validation
- Gzip compression for text assets
- Falls back to TanStack Start handler for dynamic routes

### State Management Pattern

- **Server State**: TanStack Query handles all server data fetching/caching
- **UI State**: React hooks (useState, useReducer) for local component state
- **Global State**: Context API when needed, Router context for auth state
- **Forms**: React Hook Form + Zod validation

### Environment Variables

Required for Appwrite integration (see `.env.example`):
- `APPWRITE_ENDPOINT` - Appwrite server URL
- `APPWRITE_API_KEY` - API key for server-side operations
- `APPWRITE_PROJECT_ID` - Project identifier (also used client-side)
- `APPWRITE_BUCKET_ID` - Storage bucket ID
- `VITE_INSTRUMENTATION_SCRIPT_SRC` - Optional analytics script

### Key Patterns

**Adding a New Route:**
1. Create file in `src/routes/` (e.g., `_public/about.tsx`)
2. Run `bun run generate:routes` to update route tree
3. Route is automatically typed and accessible

**Adding a Server Function:**
1. Define in `src/server/functions/` using `createServerFn()`
2. Use Zod for input validation via `.inputValidator()`
3. Import and call from client components (fully type-safe)

**Adding a Protected Route:**
1. Place file under `src/routes/_protected/`
2. Automatically requires authentication via `_protected.tsx` loader
3. Access current user via `Route.useLoaderData()` from `__root.tsx`

**Handling Forms with Server Actions:**
```tsx
const form = useForm({ resolver: zodResolver(schema) })
const onSubmit = async (data) => {
  try {
    await serverFn({ data })
  } catch (error) {
    // Handle error
  }
}
```
